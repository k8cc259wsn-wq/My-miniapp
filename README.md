# MassUp 💪 — Telegram Mini App для набора веса

Приложение для тех, кто набирает мышечную массу: дневник питания с расчётом нормы калорий и БЖУ, готовые программы тренировок и конструктор своей программы, трекер веса/замеров/фото, streak-серии и достижения.

Все данные пользователя хранятся в **Telegram CloudStorage** — облаке самого Telegram, привязанном к паре «пользователь + бот». Свой сервер и база данных не нужны.

---

## Что внутри

- 🔥 Расчёт суточной нормы калорий и БЖУ под набор массы (формула Миффлина-Сан Жеора + профицит)
- 🍽 Дневник питания с базой продуктов (БЖУ на 100 г) и историей по дням
- 💪 3 готовые программы тренировок (новичок / средний / продвинутый) + конструктор своей из библиотеки упражнений
- ⏱ Таймер отдыха между подходами во время тренировки
- 📈 График динамики веса, замеры тела, фото-прогресс
- 🏆 Streak (дни подряд с записями) и система достижений
- 🎨 Премиальный тёмный интерфейс, адаптированный под Telegram Mini Apps

---

## Технологии

- React 18 + TypeScript + Vite
- React Router (HashRouter — обязателен для Mini Apps, не требует серверного роутинга)
- Zustand для состояния приложения
- Recharts для графиков
- Telegram Web App SDK (`window.Telegram.WebApp`) + CloudStorage API

## Быстрый старт (локальная разработка)

Понадобится [Node.js](https://nodejs.org/) версии 18 или новее.

```bash
# 1. Установить зависимости
npm install

# 2. Запустить дев-сервер
npm run dev
```

Откроется `http://localhost:5173`. Вне Telegram приложение автоматически переключается на `localStorage` вместо CloudStorage — это специально сделано в `src/utils/telegram.ts`, чтобы можно было разрабатывать и тестировать в обычном браузере.

## Как опубликовать приложение и подключить к боту Telegram

### Шаг 1 — Создать бота

1. Откройте в Telegram бота [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`, придумайте имя и username (должен заканчиваться на `bot`)
3. BotFather выдаст токен — он не нужен для самого мини-аппа, но сохраните его

### Шаг 2 — Захостить приложение

Самый простой бесплатный вариант — **GitHub Pages**, и в этом репозитории уже настроен автодеплой:

1. Создайте репозиторий на GitHub и залейте туда этот проект:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПО.git
   git push -u origin main
   ```
2. В репозитории на GitHub зайдите в **Settings → Pages**, в разделе "Build and deployment" выберите **Source: GitHub Actions**
3. Workflow `.github/workflows/deploy.yml` соберёт и опубликует проект автоматически при каждом пуше в `main`
4. После первого деплоя (вкладка **Actions** покажет прогресс) приложение будет доступно по адресу:
   ```
   https://ВАШ_ЛОГИН.github.io/НАЗВАНИЕ_РЕПО/
   ```

### Шаг 3 — Подключить ссылку к боту

1. Снова откройте [@BotFather](https://t.me/BotFather)
2. Отправьте `/mybots`, выберите своего бота
3. **Bot Settings → Menu Button → Configure Menu Button**
4. Вставьте ссылку на GitHub Pages из шага 2 и название кнопки (например, "Открыть MassUp")

Готово — кнопка меню бота откроет приложение прямо внутри Telegram, и CloudStorage заработает автоматически.

## Структура проекта

```
src/
  components/      UI-компоненты, разбитые по доменам
    ui/             Базовые переиспользуемые элементы (Button, Card, BottomSheet…)
    layout/         Каркас экранов (TabBar, PageHeader, PageContainer)
    nutrition/      Компоненты дневника питания
    workout/        Компоненты тренировок
    progress/       Компоненты трекера прогресса
    charts/         Графики на recharts
  pages/            Страницы-экраны приложения (по одной на маршрут)
  store/            Zustand-стор + репозитории чтения/записи в CloudStorage
  data/             Статические данные: база продуктов, упражнения, готовые программы
  hooks/            Кастомные React-хуки (streak, таймер отдыха)
  utils/            Расчёты калорий, даты, обёртка над Telegram SDK
  types/            TypeScript-типы домена
```

## Архитектура хранения данных

Telegram CloudStorage имеет лимиты: до 1024 ключей и до 4096 байт на значение ключа. Поэтому:

- Профиль и список разблокированных достижений хранятся как один ключ
- Дневник питания, лог тренировок и записи веса **чанкуются по месяцам** (`massup_nutrition_2026-06` и т.п.) — так каждый ключ остаётся в пределах лимита, даже при активном ежедневном использовании

Вся логика чтения/записи инкапсулирована в `src/store/*Repository.ts` — компоненты не работают с CloudStorage напрямую.

## Как добавить свои продукты или упражнения

- Продукты: добавьте объект в массив `FOOD_DATABASE` в `src/data/foods.ts`
- Упражнения: добавьте объект в массив `EXERCISES` в `src/data/exercises.ts`
- Готовые программы: добавьте объект в массив `WORKOUT_PROGRAMS` в `src/data/programs.ts`
